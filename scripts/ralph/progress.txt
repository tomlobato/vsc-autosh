## Codebase Patterns
- Build: `npm run compile` (esbuild) for bundling, `npm run check-types` (tsc --noEmit) for type checking
- Entry point: `src/extension.ts` with `activate(context)` / `deactivate()` exports
- esbuild config is in `esbuild.js` (CJS, context API), externals: ['vscode']
- Settings are declared in `package.json` under `contributes.configuration.properties`
- Use `activationEvents: ['onStartupFinished']` for lazy activation
- Config reads: `vscode.workspace.getConfiguration('autosh').get<T>('key', default)` — no caching needed
- Status bar: create with `vscode.window.createStatusBarItem(Right)`, push to `context.subscriptions`
- Module pattern: `src/config.ts` for settings, `src/statusbar.ts` for status bar, `src/handlers.ts` for file handlers
- Use `WorkspaceEdit` (not `editor.edit()`) when the document isn't yet open in an editor
- `onDidCreateFiles` fires for new files only — not when opening existing files
- chmod uses `fs.promises.chmod(path, 0o755)` with try/catch, skipped on Windows
- Quick Pick: `showQuickPick(items)` returns `undefined` on cancel — use for user selection with abort
- Cursor positioning in existing docs: use `content.split('\n').length - 1` not `document.lineCount`
